## 1. 動画スタイル: ハイブリッド

場面によって画面構成を切り替える。

| 場面 | 画面構成 | 画像ソース |
|------|---------|-----------|
| 導入・タイトル | 全画面画像 + タイトルテロップ | ストック画像 or AI生成 |
| データ提示 | スライド主体 + キャラ小 | Google Slides |
| 対話・解説 | キャラ立ち絵 + 背景画像 | ストック画像 |
| まとめ・結論 | 全画面画像 + まとめテロップ | ストック画像 |

### 設計原則

- 視覚変化は「画面構成の切り替え」で確保する。個々の画像を動かすのではなく、異なるレイアウトの場面を交互に配置する
- 情報密度の高い場面（データ・手順）は静止。読者が情報を読み取る時間を確保する
- 対話場面はキャラクターのリップシンクとセリフ切替で視覚変化が生まれる

## 2. アニメーションルール (案、Phase 4 後に確定)

### 2.1 画像種別による判定

| 画像種別 | アニメーション | 理由 |
|---------|-------------|------|
| Google Slides (テキスト主体) | static | テキスト可読性が最優先 |
| ストック写真・イラスト | ken_burns (ゆっくりズーム) | 静止画の単調さを軽減 |
| AI生成画像 | ken_burns | 同上 |
| 画像なし (キャラのみ) | N/A | 背景ベースが表示される |

### 2.2 場面による判定 (上記を上書き)

| 場面 | アニメーション | 理由 |
|------|-------------|------|
| 章の開始 | zoom_in | 注目を集める |
| データ・統計提示 | static | 数値の可読性優先 |
| トピック切替 | (TBD) | フェードやクロスフェードで十分かもしれない |
| まとめ・結論 | zoom_out | 俯瞰感の演出 |

### 2.3 未決定事項

- [ ] トピック切替時にパン (pan_left/pan_right) が有効か、フェードだけで十分か → 1本作って判断
- [ ] ken_burns のズーム量 (現行 1.05) が適切か → 実機プレビューで判断
- [ ] 全画面画像の場面でアニメーションが必要か → 他の「売れてる」ゆっくり解説を参考に判断
- [ ] 画像の切替頻度 (5-15秒) は適切か → 実際の視聴体験で判断

## 3. 画面レイヤー構成

### 3.1 レイヤー配置 (SP-052 Section 2.1 準拠)

```
Layer 7: テキストオーバーレイ (章タイトル、キーポイント)
Layer 6: 字幕 (話者セリフ)
Layer 5: キャラクタースプライト (れいむ右、まりさ左)
Layer 4: 画像B (クロスフェード用偶数)
Layer 3: 画像A (クロスフェード用奇数)
Layer 2: 背景ベース (グラデーション)
Layer 1: BGM
```

### 3.2 場面別レイヤー使用

| 場面 | 使用レイヤー | 備考 |
|------|------------|------|
| 導入 | L2 + L3 + L6 + L7 | 全画面画像 + タイトル。キャラなし |
| データ提示 | L2 + L3 + L5 + L6 + L7 | スライド + キャラ小 + キーポイント |
| 対話 | L2 + L3 + L5 + L6 | キャラ + 背景画像 + 字幕 |
| まとめ | L2 + L3 + L6 + L7 | 全画面画像 + まとめテロップ |

## 4. テキストオーバーレイの使い分け

| オーバーレイ種別 | 表示条件 | 必要性 |
|---------------|---------|--------|
| chapter_title | セクション切り替わり時 | 高 -- 動画の構造を視聴者に伝える |
| key_point | key_points フィールド存在時 | 中 -- 情報密度を上げるが、過剰だと視覚ノイズ |
| statistic | 数値データ検出時 | 低 -- スライドに含まれていれば不要 |
| source_citation | 引用元がある場合 | 低 -- 信頼性向上だが、視聴テンポを妨げる可能性 |

### 未決定事項

- [ ] key_point を「常に表示」か「スライドにない場合のみ」か → 1本作って判断
- [ ] statistic オーバーレイはスライドと重複しないか → 実機確認
- [ ] オーバーレイの表示時間 (現行 4-7秒) は適切か → 視聴テスト
