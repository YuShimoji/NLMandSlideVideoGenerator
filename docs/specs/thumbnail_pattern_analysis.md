## 2. 成功パターンの体系化

### 2.1 レイアウトパターン (Layout Pattern)

#### Pattern A: 中央テキスト型 (Center Text)
```
┌──────────────────────┐
│   背景画像 (暗め)      │
│                       │
│    ★大テキスト★       │
│   (赤/黄、縁取り)      │
│    サブテキスト         │
│                       │
└──────────────────────┘
```
- **用途**: ミステリー、謎、総集編
- **代表例**: 「世界の謎72選」「南極の謎5選」
- **特徴**: テキストが画面の40-60%を占有。背景は暗くてテキストが際立つ
- **再生数**: 100万+ 多数

#### Pattern B: 左画像+右テキスト型 (Left Image + Right Text)
```
┌──────────┬───────────┐
│          │           │
│  人物/   │  ★テキスト│
│  建造物  │   (大)    │
│  画像    │  サブテキ  │
│          │   スト    │
└──────────┴───────────┘
```
- **用途**: 時事、人物解説、歴史
- **代表例**: ゆっくり防衛チャンネルの国際情勢解説
- **特徴**: 人物の切り抜き+テキストで情報と感情を同時に伝達
- **再生数**: 5万-10万級

#### Pattern C: 地図+矢印型 (Map + Arrow)
```
┌──────────────────────┐
│  地図背景              │
│      ←赤矢印→        │
│                       │
│  「なぜXなのか？」     │
│  (白テキスト+赤縁取り) │
└──────────────────────┘
```
- **用途**: 地理、地政学、インフラ
- **代表例**: 地理ゆっくり解説の高再生数動画
- **特徴**: 矢印で視線誘導、地図で「知的好奇心」を刺激
- **再生数**: 50万-98万級

#### Pattern D: 縦分割型 (Vertical Split)
```
┌─────┬─────┬─────┬─────┐
│画像1│画像2│画像3│画像4│
│     │     │     │     │
│─────┼─────┼─────┼─────│
│見出1│見出2│見出3│見出4│
│小文字│小文字│小文字│小文字│
│     │     │     │     │
├─────┴─────┴─────┴─────┤
│  ★メインテキスト★     │
│  (赤、大きく)          │
└───────────────────────┘
```
- **用途**: 複数テーマ対比、哲学的問い、「X vs Y vs Z」
- **特徴**: 全文を読ませるためではなく、「複数の視点がある」ことを視覚的に伝える確立されたパターン。個々の小テキストの可読性より全体の情報密度が重要
- **再生数**: パターン適用時は高い。ただし微調整が大きく影響
- **制御ポイント**: 各セクションの画像品質、色の統一性、メインテキストの可読性

#### Pattern E: 数字+リスト型 (Number + List)
```
┌──────────────────────┐
│  背景画像              │
│                       │
│   ★XX選★             │
│   (巨大な数字)         │
│   サブタイトル          │
│                       │
└──────────────────────┘
```
- **用途**: 総集編、まとめ、ランキング
- **代表例**: 「世界の謎72選」「パラドックス5選」
- **特徴**: 数字が最大のフック。「30選」より「72選」の方が知的好奇心を刺激
- **再生数**: 100万+ (総集編効果)

### 2.2 色彩パターン (Color Pattern)

| パターン名 | 背景 | メインテキスト | アクセント | 用途 |
|-----------|------|--------------|-----------|------|
| Dark Red | 暗い画像/黒 | 赤 (#FF0000-#CC0000) | 白縁取り | 時事、ミステリー、衝撃系 |
| Dark Yellow | 暗い画像/黒 | 黄 (#FFFF00-#FFD700) | 赤縁取り | 科学、雑学、発見系 |
| Map Blue | 地図背景 | 白 (#FFFFFF) | 赤矢印 | 地理、インフラ |
| High Contrast | 暗青/暗紫 | 白+赤 | シアン/緑 | テクノロジー、AI |
| Warm Alert | 暗赤 | 黄+白 | オレンジ | 緊急ニュース、警告系 |

### 2.3 テキストパターン (Text Pattern)

#### フック構文 (Hook Formula)

| パターン | 例 | 効果 |
|---------|-----|------|
| なぜX？ | 「なぜ船が通過できない？」 | 知的好奇心の直接刺激 |
| Xの正体 | 「不可解な一致の正体」 | 謎の提示+解決の約束 |
| Xの謎 | 「南極の謎5選」 | ミステリー感の醸成 |
| 絶対にX | 「絶対に矛盾してしまう」 | 断定による強い主張 |
| 衝撃のX | 「衝撃の結末を迎えた」 | 感情的インパクト |
| XX選/XX個 | 「世界の謎72選」 | ボリュームの約束 |

#### テキスト階層

| レベル | サイズ比 | 文字数目安 | 役割 |
|--------|---------|----------|------|
| メインテキスト | 100% (72-96pt相当) | 5-12文字 | 視覚的フック、最初に目に入る |
| サブテキスト | 50-60% | 10-25文字 | 補足情報、クリック判断材料 |
| ラベル | 40% | 3-8文字 | カテゴリ表示（総集編、ゆっくり解説等） |

### 2.4 失敗パターン (Anti-Pattern)

| Anti-Pattern | 問題 | 具体例 |
|-------------|------|--------|
| テキスト過多 | 1920x1080に詰め込みすぎ、モバイルで読めない | 地理チャンネル最新動画 |
| コントラスト不足 | 背景と文字色が近い、縁取りなし | 映画レビュー系 |
| 小さいフォント | メインテキストが画面の20%以下 | 伸び悩みチャンネル全般 |
| 汎用ストック画像 | テーマとの関連性が弱い | PIL自動生成サムネイル |
| 複雑な構図 | 視線が散る、焦点が定まらない | 情報過多な4分割(制御不足時) |
| ブランド一貫性なし | 動画ごとにスタイルがバラバラ | 小規模チャンネル |

## 3. YMM4テンプレート設計への応用

### 3.1 テンプレートカテゴリ

上記パターンA-Eに対応するYMM4テンプレートを用意する。
各テンプレートは以下の要素で構成:

| 要素 | YMM4 Item Type | プレースホルダー | 制御対象 |
|------|---------------|-----------------|---------|
| 背景画像 | ImageItem | `{{BACKGROUND}}` | テーマ画像、地図等 |
| メインテキスト | TextItem | `{{MAIN_TEXT}}` | フック文言 |
| サブテキスト | TextItem | `{{SUB_TEXT}}` | 補足説明 |
| アクセント画像 | ImageItem | `{{ACCENT_IMAGE}}` | 人物切り抜き、矢印等 |
| オーバーレイ | ImageItem | `{{OVERLAY}}` | グラデーション、枠等 |
| ラベル | TextItem | `{{LABEL}}` | 「ゆっくり解説」等 |

### 3.2 制御可能なパラメータ

サムネイルの再生数への影響が大きいパラメータを、パイプラインから制御可能にする:

| パラメータ | 影響度 | 制御方法 |
|-----------|-------|---------|
| メインテキスト文言 | 極大 | `{{MAIN_TEXT}}` プレースホルダー差し替え |
| テキスト色 | 大 | TextItem.FontColor 差し替え (JSON直接) |
| 背景画像 | 大 | `{{BACKGROUND}}` 差し替え |
| テキストサイズ | 中 | TextItem.FontSize 差し替え |
| 縁取り色・太さ | 中 | TextItem.OutlineColor/OutlineWidth |
| テキスト位置 | 中 | TextItem.X/Y 座標 |
| 影の有無 | 小 | TextItem.ShadowColor |

### 3.3 品質差異の制御

同じレイアウトでも再生数に大きな差が出る「微細な違い」を制御するため、以下のバリエーション軸を定義:

| 軸 | 選択肢 | 効果 |
|-----|--------|------|
| テキスト縁取り | 白/黒/赤/なし | 可読性とインパクトのバランス |
| 背景暗度 | 原画/50%暗化/70%暗化 | テキスト可読性に直結 |
| フォントウェイト | Regular/Bold/Extra Bold | 太いほどインパクト大だが品がなくなる |
| テキスト回転 | 0度/微傾斜(-3~3度) | 動的感の演出 |
| 色温度 | 寒色/暖色/ニュートラル | テーマとの一致 |

## 4. パイプライン統合設計

### 4.1 ワークフロー

```
台本 (script.json)
    ↓
サムネイル文言生成 (Gemini or 手動)
    ↓
テンプレート選択 (パターンA-E)
    ↓
パラメータ構成 (色/画像/テキスト)
    ↓
YMM4 .y4mmp 出力
    ↓
YMM4で開いて1フレーム書き出し
    ↓
人間レビュー + 微調整
    ↓
最終 thumbnail.png (1920x1080)
```

### 4.2 Geminiによるサムネイル文言生成

台本内容からサムネイル用のフック文言を自動生成する。
成功パターンのフック構文 (Section 2.3) をプロンプトに組み込む。

```
入力: 台本タイトル + 最初の3セグメント
出力: {
  "main_text": "なぜXは...",     // 5-12文字
  "sub_text": "驚きの理由が...",  // 10-25文字
  "label": "ゆっくり解説",       // 固定or選択
  "suggested_pattern": "C",      // 推奨レイアウトパターン
  "suggested_color": "dark_red"  // 推奨色パターン
}
```

### 4.3 A/Bテスト対応

同一動画に対して複数のサムネイルバリエーションを生成し、
YouTube Studio のA/Bテスト機能に対応する:

- 同一テンプレートで文言違い (2-3パターン)
- 同一文言で色違い (2パターン)
- 異なるテンプレートパターン (2パターン)

### 4.4 CLI統合

```bash
# 基本
python scripts/research_cli.py pipeline --topic "..." --generate-thumbnail

# テンプレート指定
python scripts/research_cli.py pipeline --topic "..." --thumbnail-template center_text

# バリエーション生成
python scripts/research_cli.py pipeline --topic "..." --thumbnail-variants 3

# テンプレート一覧
python scripts/research_cli.py thumbnail-templates
```

## 5. テンプレート作成ガイド (YMM4)

### 5.1 プロジェクト設定

- 解像度: 1920x1080 (16:9)
- フレームレート: 30fps
- 長さ: 1フレーム (0.033秒)
- YMM4のサムネイル書き出しオプションを使用

### 5.2 Pattern A テンプレート作成手順

1. YMM4で新規プロジェクト作成 (1920x1080)
2. ImageItem を配置 → FilePath に `{{BACKGROUND}}`
   - サイズ: 1920x1080 (全画面)
3. ShapeItem (矩形) を配置 → 半透明黒オーバーレイ
   - 色: rgba(0, 0, 0, 0.4-0.6)
   - サイズ: 1920x1080
4. TextItem を配置 → Text に `{{MAIN_TEXT}}`
   - フォント: Noto Sans JP Black (or 源ノ角ゴシック Heavy)
   - サイズ: 80-96pt
   - 色: #FF0000 (赤)
   - 縁取り: 白、4-6px
   - 影: 黒、offset 3px
   - 位置: 中央やや下
5. TextItem を配置 → Text に `{{SUB_TEXT}}`
   - フォント: Noto Sans JP Bold
   - サイズ: 36-48pt
   - 色: #FFFFFF (白)
   - 縁取り: 黒、2-3px
   - 位置: メインテキストの上
6. config/thumbnail_templates/center_text.y4mmp として保存

### 5.3 Pattern D (縦分割) テンプレート作成手順

1. 基本プロジェクト設定 (同上)
2. ImageItem x4 を等幅配置
   - 各 480x540 (上半分)
   - FilePath: `{{IMAGE_1}}` ... `{{IMAGE_4}}`
3. ShapeItem x4 → 各画像の下にラベル背景
   - 色: テーマカラー (各セクション異なる色)
4. TextItem x4 → 各セクション見出し
   - Text: `{{SECTION_1}}` ... `{{SECTION_4}}`
   - サイズ: 32-40pt
5. TextItem x4 → 各セクション小テキスト
   - Text: `{{DETAIL_1}}` ... `{{DETAIL_4}}`
   - サイズ: 18-24pt
6. TextItem → メインテキスト (下部帯)
   - Text: `{{MAIN_TEXT}}`
   - サイズ: 72-88pt、赤、縁取り白
7. config/thumbnail_templates/vertical_split.y4mmp として保存

## 6. 品質チェックリスト

サムネイル生成後、以下を確認:

- [ ] モバイル表示 (小サイズ) でメインテキストが読めるか
- [ ] メインテキストが12文字以内か
- [ ] テキストと背景のコントラスト比が十分か
- [ ] 赤/黄/白のアクセント色が使われているか
- [ ] 視線の焦点が1箇所に定まるか
- [ ] 同チャンネルの他動画と視覚的一貫性があるか
- [ ] フック構文 (なぜ/の正体/の謎/XX選) が使われているか
- [ ] 背景画像がテーマと関連しているか
