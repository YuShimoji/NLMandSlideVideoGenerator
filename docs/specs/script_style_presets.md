# 台本スタイルプリセット仕様 (SP-036)

**最終更新**: 2026-03-17
**ステータス**: done

---

## 1. 目的

動画種別ごとにGeminiプロンプトをテンプレート化し、台本品質の一貫性を構造的に担保する。

### 1.1 現状 (As-Is)

- `gemini_integration.py` に単一の固定プロンプトがハードコード
- 「YouTube解説動画のスクリプト作成の専門家」という固定ロール
- 動画種別（ニュース/解説/まとめ等）による構成・トーンの切り分けなし
- 話者構成が `"speaker": "Host"` 固定

### 1.2 目標 (To-Be)

- 動画種別ごとにプリセット（プロンプトテンプレート）を定義
- CLIの `--style` オプションまたはWeb UIのドロップダウンで選択
- プリセットはJSONファイルで管理し、ユーザーがカスタマイズ可能
- 既存のプロンプト構造（JSON出力形式）は変更しない

---

## 2. プリセット定義

### 2.1 ファイル構成

```
config/
  script_presets/
    news.json          # ニュース/時事解説
    educational.json   # 解説/教育系
    summary.json       # まとめ/ランキング系
    default.json       # 汎用（現行プロンプト互換）
```

### 2.2 プリセットスキーマ

```json
{
  "name": "news",
  "display_name": "ニュース/時事解説",
  "role": "あなたはYouTubeニュース解説動画のスクリプト作成の専門家です。",
  "tone": "客観的かつ分かりやすい報道調",
  "structure": [
    "速報の概要（何が起きたか）",
    "背景と経緯",
    "各立場の見解",
    "今後の見通し",
    "まとめと視聴者への示唆"
  ],
  "requirements": [
    "情報源を明示してください",
    "速報性を意識し、最新の状況を優先してください",
    "複数の視点を提示してください",
    "憶測と事実を明確に区別してください"
  ],
  "speakers": {
    "default_count": 2,
    "default_names": ["Host1", "Host2"],
    "style": "対談形式（メインキャスター+解説者）"
  },
  "segment_density": {
    "short_5min": "5-7",
    "medium_15min": "12-18",
    "long_30min": "25-35"
  }
}
```

### 2.3 初期プリセット（4種）

| プリセット | ロール | トーン | 構成パターン | 話者スタイル |
|-----------|--------|--------|-------------|-------------|
| `default` | 解説動画の専門家 | 分かりやすい話し言葉 | 導入→本論→まとめ | Host (1名) |
| `news` | ニュース解説の専門家 | 客観的な報道調 | 速報→背景→見解→見通し→まとめ | Host1+Host2 (対談) |
| `educational` | 教育コンテンツの専門家 | 丁寧で体系的 | 前提知識→概念説明→具体例→演習→復習 | 先生+生徒 (2名) |
| `summary` | エンタメまとめの専門家 | テンポの良い語り口 | ランキング/リスト形式 | ナレーター (1名) |

## 3. 実装計画

### Phase 1: プリセット読み込み基盤

| 項目 | 内容 |
|------|------|
| 対象 | `src/notebook_lm/gemini_integration.py` |
| 変更 | プロンプト構築部分をプリセットJSON駆動に変更 |
| 後方互換 | `--style` 省略時は `default.json` = 現行プロンプトと同一出力 |
| テスト | プリセット読み込み + プロンプト構築 + 既存テスト維持 |

### Phase 2: CLI/UI統合

| 項目 | 内容 |
|------|------|
| CLI | `research_cli.py pipeline --style news` |
| Web UI | material_pipeline.py にスタイル選択ドロップダウン追加 |
| バリデーション | 不明なスタイル名はエラー、利用可能なスタイル一覧表示 |

---

## 4. 設計判断

| 判断 | 内容 | 理由 |
|------|------|------|
| JSONファイル管理 | プリセットはJSONで外部化 | ユーザーが追加・編集可能にする |
| 既存出力形式維持 | JSON出力スキーマは変更しない | CsvAssembler/YMM4プラグインへの影響を排除 |
| 段階的拡張 | まず4プリセットから開始 | 実運用フィードバックで追加 |

---

## 5. 品質軸との対応

| 品質軸 | SP-036の貢献 |
|--------|-------------|
| 制作スピード | プリセット選択だけで適切なプロンプトが生成される |
| 情報密度/正確性 | 種別ごとの要件（ソース明示、多角的視点等）をプロンプトに埋め込む |
| 視覚的完成度 | - (直接的な寄与なし) |
| 一貫性/再現性 | 同一プリセットなら同一品質の台本が生成される |
