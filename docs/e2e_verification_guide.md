# E2E 手動検証ガイド

最終更新: 2026-03-13

CSV入力からYMM4での最終動画出力までのE2Eフローを手動で検証する手順書。

---

## 前提条件

- Python 3.11 + venv 環境構築済み
- YMM4 インストール済み（NLMSlidePlugin デプロイ済み）
- サンプルCSVファイル（後述）

## Step 1: サンプルCSV準備

`Speaker,Text` 形式のCSVを用意する。

```csv
Speaker1,本日のテーマはAI技術の進化です
Speaker2,2026年のAI市場は急成長しています
Speaker1,では具体的なデータを見てみましょう
Speaker2,AIの導入率は42%に達しました
Speaker1,まとめると今後も成長が続く見込みです
```

ファイル名例: `test_e2e.csv`

## Step 2: Web UI でYMM4プロジェクト生成

```powershell
# プロジェクトルートで実行
streamlit run src/web/web_app.py
```

1. ブラウザで表示されるWeb UIの「CSV Pipeline」ページを開く
2. CSVファイルをアップロード
3. 「エクスポート」を実行
4. 完了メッセージを確認

### 検証ポイント

| 確認項目 | 期待値 | 確認方法 |
|----------|--------|----------|
| プロジェクトディレクトリ | `data/ymm4/ymm4_project_YYYYMMDD_HHMMSS/` が生成 | エクスプローラで確認 |
| project.y4mmp | 存在する | ファイル確認 |
| timeline_plan.json | CSVの行数と同数のセグメント | JSONを開いて確認 |
| slides_payload.json | 存在する | ファイル確認 |
| text/timeline.csv | 元CSVと内容一致 | diffまたは目視 |
| render_metadata.json | 存在する | ファイル確認 |

## Step 3: YMM4 でCSVインポート + Voice自動生成

1. YMM4を起動
2. メニュー「NLMSlidePlugin」 > 「CSVタイムラインをインポート」
3. CSVインポートダイアログが表示される
4. 「参照...」でStep 2で生成された `text/timeline.csv` を選択
5. 「音声を自動生成（ゆっくりボイス）」チェックボックスがONであることを確認（デフォルトON）
6. 「字幕を追加」チェックボックスを必要に応じて設定
7. 「プレビュー」ボタンでCSV内容を確認（行数・話者・テキスト）
8. 「インポート」ボタンを実行
9. Voice生成の進捗バーが進行し、完了メッセージが表示されることを確認

### 検証ポイント

| 確認項目 | 期待値 | 確認方法 |
|----------|--------|----------|
| プレビュー表示 | CSVの行数と一致、話者・テキストが正しい | ダイアログ内DataGrid |
| Voice Speaker検出 | 「Found N voice speaker(s)」がログに表示 | ダイアログ内ログタブ |
| Voice生成進捗 | 各セグメントで進捗バーが進行 | 進捗バー・ステータス表示 |
| 完了メッセージ | 「Import Complete!」+ 生成数/スキップ数/失敗数 | ダイアログ |
| タイムライン上のアイテム | CSVの行数と一致 | YMM4タイムライン目視 |
| テキスト内容 | CSV各行のText列と一致 | 各アイテムをクリックして確認 |
| 音声プレビュー | 正常に再生 | YMM4で各セグメント再生 |
| エラー | なし（またはログに記録） | ダイアログ内ログタブ + `%LOCALAPPDATA%\NLMSlidePlugin\logs\csv_import_runtime.log` |

### Voice Speaker未検出時のフォールバック

Voice Speakerが見つからない場合、自動的にVoiceなしインポートにフォールバックする。
この場合は手動でゆっくりボイスを割り当てる:

1. タイムライン上の各テキストアイテムを選択
2. 音声を「ゆっくり霊夢」「ゆっくり魔理沙」等に割り当て
3. 各セグメントで音声プレビュー再生

## Step 5: 最終レンダリング

1. YMM4のタイムラインで全体プレビュー
2. 「ファイル」>「動画出力」（またはCtrl+Shift+E）
3. 出力先と設定を確認して実行
4. レンダリング完了を待つ

### 検証ポイント

| 確認項目 | 期待値 | 確認方法 |
|----------|--------|----------|
| mp4ファイル | 指定先に生成 | エクスプローラで確認 |
| 動画再生 | 正常に再生可能 | メディアプレイヤーで開く |
| 解像度 | 1920x1080 (16:9) | プロパティまたはプレイヤー情報 |
| 音声同期 | テキストと音声が同期 | 視聴確認 |
| セグメント数 | CSV行数と一致 | 目視カウント |

## 既知の制限事項

- Voice Speaker未検出時は手動での音声割り当てが必要（VoiceSpeakerDiscovery 3層フォールバック後）
- AutoHotkeyスクリプトはPoC段階。手動操作を推奨
- YMM4のレンダリング設定（ビットレート、コーデック等）はユーザーの環境に依存
