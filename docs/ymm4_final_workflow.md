# YMM4 最終ワークフロー標準化手順 (Standardized Final Workflow)

**最終更新**: 2026-03-01
**ステータス**: 運用版

本ドキュメントは、Python側の動画生成パイプラインからYMM4へ引き継ぎ、最終的なMP4出力に至るまでの**標準化されたプロセス（最も安全で確実な手動運用・半自動運用）**と、エラー発生時の**リカバリー手順**を定めたものです。

---

## 1. ワークフロー概要

本プロジェクトでは、YMM4への完全な「不可視バックグラウンド出力」を目指していますが、Windows環境やYMM4の仕様によって出力に失敗するリスクが存在します。
そのため、**必ずYMM4のGUIを開き、プロジェクト（.y4mmp）の整合性を確認してからMP4を書き出す**手順を標準とします。

### 全体フロー

1. **[Python] パイプライン実行** (`run_csv_pipeline.py` or Web UI)  
   出力物: `project.y4mmp`, 音声ファイル群, `ymm4_automation.ahk`
2. **[YMM4 / AHK] プロジェクトの読み込みと確認**  
   AutoHotkey スクリプトを実行するか、手動でYMM4を開き、生成されたプロジェクトを開く。
3. **[YMM4 / AHK] 音声ファイル一括インポート**  
   YMM4のファイルインポート機能（`Ctrl+Shift+I`）などを利用して同期された音声を挿入。
4. **[YMM4] 最終確認とエンコード**  
   動画出力（`Ctrl+Shift+E`）を実行し、MP4を生成。
5. **[Python / PS] ポスト処理** (`RUN_AFTER_YMM4.ps1`)  
   生成されたMP4を指定の完了ディレクトリへ移動し、サムネイル等と結合して公開準備を整える。

---

## 2. 標準オペレーション手順 (SOP)

### Step 1: パイプライン実行

Web UI、または以下のコマンドを使用して CSV から YMM4用のプロジェクトファイルを生成します。

```powershell
python scripts/run_csv_pipeline.py --csv data/timeline.csv --audio-dir data/audio/timeline01 --export-ymm4
```

実行後、`data/ymm4/ymm4_project_YYYYMMDD_HHMMSS/` にプロジェクトファイル群が生成されていることを確認します。

### Step 2: AutoHotkey (AHK) による半自動エクスポート（推奨）

生成されたディレクトリ内にある `ymm4_automation.ahk` を実行します。

```powershell
# AHKスクリプトをダブルクリック、またはコマンドで実行
AutoHotkey.exe data/ymm4/ymm4_project_YYYYMMDD_HHMMSS/ymm4_automation.ahk
```

- AHKがYMM4を起動し、プロジェクトを読み込みます。
- 音声の差し込みと動画書き出しダイアログの手前まで自動化されます。
- **注意**: マウスやキーボードを操作するとAHKの自動操作と競合するため、操作中はPCに触れないでください。

### Step 3: 手動での動画書き出し（フォールバック）

AHKが失敗した場合、または手動操作を好む場合は以下の手順で実行します。

1. `project.y4mmp` を YMM4 で開く。
2. タイムライン上のプレースホルダー領域に、`audio/segments/` フォルダ内の音声ファイルをドラッグ＆ドロップ（またはファイルインポート）。
3. 音声とテキストが同期していることを確認（プレビュー再生）。
4. `ファイル > 動画出力`（Ctrl+Shift+E）から指定形式で出力。

### Step 4: 出力後処理 (`RUN_AFTER_YMM4.ps1`)

YMM4での出力が完了したら、作業内容の整理とデプロイ準備のために後処理スクリプトを実行します。

```powershell
.\scripts\RUN_AFTER_YMM4.ps1 -ProjectDir "data/ymm4/ymm4_project_YYYYMMDD_HHMMSS"
```

このスクリプトは以下の処理を行います。

- エクスポートされた動画(MP4)を `data/videos/` へ移動・リネーム
- プロジェクトの不要なテンポラリファイルをクリーンアップ
- 公開用のサムネイル・台本(CSV)と動画の整合性を確認

## 3. エラー時リカバリ手順 (Troubleshooting)

| エラーの事象 | 発生する段階 | 原因と対処法 |
|-------------|--------------|--------------|
| **AHK実行時にフォーカスが外れて操作が失敗する** | Step 2 | AHKのUIキー操作はデスクトップ環境に依存します。一度AHKプロセスを終了し、SOPの「Step 3: 手動での動画書き出し」へ移行してください。 |
| **YMM4読み込み時にファイルパス不足・エラーが出る** | Step 2 / 3 | `project.y4mmp` と同じディレクトリに `audio` や `slides_payload.json` が欠損している可能性があります。再度パイプライン(Step 1)を流し直してください。 |
| **音声とテロップのタイミングがずれる** | Step 3 | 現在のYMM4テンプレートの差分吸収に失敗した可能性があります。YMM4 GUI上でセグメントをドラッグして微調整するか、テキストの長さをCSV上で調整し再度パイプラインを実行します。 |
| **動画出力(エンコード)が異常に遅い、またはフリーズ** | Step 3 | PCのリソース不足か、YMM4の一時的な不具合です。YMM4を再起動し、再度プロジェクトを開いて出力のみを試みてください。 |

## 4. 将来の完全自動化へ向けて

現在、YMM4プラグインとしての .NET API 連携を視野に入れた調査が進められていますが、YMM4はデスクトップGUIを前提とした仕様のため、完全にバックグラウンドで動画生成を行うにはMoviePyといったPython製CLIバックエンド（`ExportFallbackManager` に実装済み）を選択する方が安定します。

したがって：

- **高品質な YMM4 独自の演出** を求める場合 → **本ドキュメントの半自動フローを適用**。
- **完全自動・大量生産** を求める場合 → **Settings から Editing Backend を `moviepy` に変更**して運用してください。
